자료구조 실습내용 정리

구분 개인프로젝트
개발기간 3개월
사용기술 Language: C
Github, Naver Blog

[개요]

편입 첫학기에 들었던 자료구조 과목의 정리노트이다. 코딩실력이 한참 부족할 때지만, 나만의 작업물을 만들고 싶은 열망으로 시작했다. 각 주차별 코드는 깃허브에, 상세한 풀이과정은 네이버 블로그에 서술해 두었다.
링크URL: [Github] [Naver blog]

[개발내용]

개념을 익히는 문제들은 다음과같이 풀이과정이 남는다. 각 주차마다 1개에서 많게는 10개남짓의 문제가 과제로 출제되었다. 대부분 표의 빈칸에 문자를 채워넣거나, 자료구조를 그림으로 그려서 구현하는등의 문제였다. 나는 파워포인트를 사용해 풀이과정 이미지를 만들고 블로그에 글로써 서술해 풀이과정을 남겼다. 중요한 개념을 배울때마다 각 주차에는 항상 코딩과제가 있었다. 코딩과제는 대부분 교재의 코드에서 특정 부분의 기능을 향상하거나 또는 그 주차의 자료구조를 활용하여 간단한 동작을 하는 프로그램을 구현하는것이었다. 다음과같이 중간과정을 서술해야 하는 문제는 풀이가 매우 길어졌다(이하생략)

[느낀점]

수업이 점점 진행되면서 앞에서 배웠던 내용을 뒤에서도 써야할 때가 있었는데, 그 때마다 강의노트나 교과서를 다시 보지 않고 내가 사용했던 코드나 풀이를 보면서 많은 도움을 얻을 수 있었다. 나는 자료구조 수업을 듣기 전에 "생활코딩"에서 베포한 Web 강의를 듣고 html을 배웠었는데, 처음으로 블로그 포스팅을 하면서 웹 기반의 문서편집기가 가진 기능들이 정말 대단하고 신기한 기능이 많다고 느꼈다. 필요에 따라서 코드와 이미지, 글을 복합적으로 표현하고 스타일을 지정할수 있다는게 내가 처음으로 컴퓨터 공학 공부를 시작하면서 프로그램의 원리에 대해 생각해보고 감탄했던 순간이었다. 특히 Web 기반의 문서는 언제 어디서나 접근이 가능해서, 내가 자료구조 기말고사를 보러가는 동안에도 스마트폰으로 요약노트를 보며 공부할수 있다는게 혁신적이었다.

지금은 C언어를 사용한 프로그래밍을 자주 하지 않아 다른 문서들을 참고하지만, 각 자료구조들에 대한 개념과 원리를 이 떄 잘 정리해놓고 반복해서 보았기에 현재 조금이나마 적재적소에 알맞은 자료구조를 사용할 수 있게 되었다고 느낀다.

첫 작품(?)이기에 많이 부족하고 어디 내놓기 부끄럽지만 한편으론 뿌듯하다.

자동 ERP 매크로

구분 개인프로젝트(인턴)
개발기간 2개월
사용기술 Language: Python
Module: selenium, beautifulsoup, pyautoGUI
기타 산학 보안협약에 따라 private repository로 운용

[개요]

ERP 시스템의 발주 프로세스를 자동화하기 위한 매크로 프로그램이다.

[배경]

자사의 ERP 시스템의 중계자로서 고객이 주문을 요청하면 자사가 이를 확인하고 ERP프로그램, 웹사이트를 통해 배송요청을 진행해야했다. 기존엔 고객사와 납품사의 ERP시스템이 다르기 때문에 연동하는 부분이 인력으로 이루어지고 있었다. 따라서 매크로는 고객사 주문접수, 납품사 배송요청을 자동으로 연계하고자 설계되었다.

[개발내용]

매크로의 실행화면이다. Python이 설치된 환경의 컴퓨터에서 Cmd창을 통해 실행된다. 매크로의 동작은 크게 3가지로 구분된다. 차례대로 다음과 같다.

1. 오늘의 주문내역 가져오기(크롤링, txt파일에 쓰기)
고객사는 물품을 요청한 주문신청들을 받아서 관리하는 홈페이지가 있다. 이 홈페이지에서 selenium 모듈을 통해 하루치의 주문신청을 크롤링 한후 txt파일에 쓰도록 구성했다.

(고객사 A사의 홈페이지)
2. 주문내역에 따라 배송요청하기(크롤링, txt파일을 읽기)
이전 단계에서 저장해둔 txt파일을 읽어들여 납품사에 배송요청을 넣는부분이다.

웹페이지로 업무를 보는 X 납품사는 1단계와 동일하게 selenium 모듈을 사용했고, 페이지간 이동과 주문내용의 기입을 크롤링 기능으로 구현할 수 있었다.
(납품사 X사의 홈페이지)
ERP 프로그램으로 업무를 보는 Y 납품사는 기존의 웹 크롤링 기능을 사용할 수 없었다. 나는 실현가능한 대안으로 이미지 인식과 키보드 제어를 활용하는 pyautoGUI 모듈을 사용하기로 했고, 크롤러와 최대한 비슷하게 동작하도록 만들었다.
(납품사 Y사의 프로그램)
3. 배송요청 확인서와 주문내역을 대조하여 기록남기기(더블체킹)
최종적으로 프로젝트의 기능검토를 받고나서, 기록을 검토하여 정상동작을 확인할 수 있도록 검수기능을 추가하라는 피드백을 받았다.

아래는 해당 부분을 구현한 사진이다.
실제 주문서와 처리기록을 대조하여 수량,물품코드,이름이 모두 동일하면 <성공> 이라고 기록하게끔 구현했다.

[도전적 이슈&느낀점]

내가 사용가능한 프로그래밍 언어가 C와 Java밖에 없는 상황에서 Python언어를 처음으로 배워가며 시작했던 프로젝트이다. 업무로써 진행하는 첫 프로젝트인데 새로운 언어를 공부하며 매주 발전된 결과를 내야 한다는것이 조금 부담도 되었던 것 같다.

지금 돌이켜보면 실패의 부담이 없고 정말 쉬운 프로젝트였는데 괜한 부담을 가지고 힘을 쏟았구나 많이 느끼게 되는 프로젝트이다. Python에 입문하는데 큰 도움이 되었고, 매주 선임개발자 분에게 코드리뷰를 받으면서 개선할점을 들을 수 있어서 정말 좋은 경험이었다.

How to use Django

개요
구분 개인프로젝트
개발기간 4주 (2020.11 ~ 12)
소개 오픈소스 소프트웨어인 파이썬의 장고 프레임워크 사용법을 안내하고 간단한 게시판 페이지를 만드는 실습교본을 제작한다.
사용기술
  1. Language: Python

  2. Framwork: Django

  3. Html, Javascript

  4. Markdown
기능 깃허브의 Readme.md는 장고 프레임워크의 정의와 사용방법, 그리고 게시판 실습코드에 대한 자세한 안내서이다.

첫 부분에서는 프레임워크의 정의와 장고 프레임워크의 구조를 서술한 동영상을 제공한다. 중반부에는 코드에 대한 설명과 구현순서가 서술되어 있다. 후반부에는 실습을 마쳤을때 구현된 웹페이지의 전체 시연영상을 제공한다.

모든 파일은 깃허브의 public repository로 공개되며 CC 라이선스를 가진다.
개발내용
1. 프레임워크의 정의와 장고 프레임워크의 구조를 서술한 문서제작
2. 장고를 사용해 CRUD 기능을 가진 게시판 제작
3. 개념설명 및 시연동영상 제작
얻은것
도전을 즐기기 본 프로젝트는 내가 장고 프레임워크를 배우고 싶어서 시작한 프로젝트 이다. 언젠가 해봐야지 하고 미루다가 과제를 기회삼아 도전해보았다. 처음 다뤄보는 프레임워크지만, 평소 웹프로그래밍과 파이썬언어에 관심이 많아서 배워가는것은 어렵지 않았다. 앞으로도 나를 성장시킬 도전들에 주저하지 않을 용기를 주는 경험이었다.
프레임워크의 활용 프레임워크를 사용해본적은 있지만, 정확하게 어떤 유용성때문에 쓰이는것이고 내가 무엇을 해야지 프레임워크를 효율적이게 사용할 수 있는지 생각하면서 활용해보지 못했다. 이번 프로젝트를 하면서 배운 프레임워크 활용에 대한 개념과 원리(?)가 많은 도움이 되었다. 앞으로 프레임워크 단위의 프로그래밍에 도전할 의욕이 샘솟는다.
사용법 문서의 작성 내가 알고있는것을 잘 설명하는것이, 잘 배우는것 보다 어렵다는 말을 역력히 체감한 프로젝트였다. 코딩 실습이나 예제들을 구글링할 때는 자료들이 왜이리 불친절하고 설명을 날림으로 할까 불만이 많았는데, 직접 서술하는 입장이 되니까 모든 경우에 대해 설명하는게 정말 힘들었다. 자세하게 기술하기 위해서 최대한 노력했지만, 아쉬운점 도 많은 프로젝트였다. 앞으로도 내가 아는 사실을 타인에게 문서로 전달해야 할 때, 어떤 태도와 생각으로 임해야 하는지 알게되었다.

Risk Predict Machine

개요
구분 팀프로젝트
개발기간 8주 (2020.10 ~ 12)
소개 도로교통 공단에서 제시하는 도로위험도지수 도출 모델을 기계학습을 통해 예측해보고 대체할만한 정확도를 가지도록 개선한다.
사용기술 1. Language: Python
2. Jupyter, Google Colab
기능 도로교통공단에서 수집한 교통사고 데이터를 기반으로 전국에 위치한 도로의 도로위험도값을 예측한다.

얻은것
유연한 계획관리 학습데이터를 구하기 위해서 전국의 사고데이터를 모두 사용하려 했다. 하지만 우리가 생각했던것 보다 API 서버의 트래픽환경이 매우 안좋았고, 일주일을 투자해도 한개 "경기도"의 모든 데이터를 얻기가 어려웠다. 우리는 협의를 통해 임의로 산발적이게 전국의 사고데이터를 목표 갯수까지만 확보하기로 하였고, 결국 그렇게 확보한 데이터로 기대 이상의 정확도를 얻을 수 있었다. 만약 처음의 문제에 매달려 있었다면 우리팀은 완성단계에 이르지 못했을수도 있었다고 생각이든다.
머신러닝의 응용 프로젝트 구현을 위해 데이터마이닝을 하고, 모델분석과 사용경험등을 쌓으면서, 타 어플리케이션 제작에서 응용시킬 수 있는 머신러닝 기술을 얻었다.
주석의 중요성 우리 팀은 초기에 스켈레톤 코드를 작성해 정돈된 코딩을 도모했으나, 분할 작업을 하면서 전체코드가 이따금씩 동작하지 않을때가 있었다. 매번 달라진 파트를 체크해서 오류점을 분석하는 어려움을 겪으면서, 코드의 용도와 의도를 잘 나타내는 주석이 필수라는 점을 깨달았다.
